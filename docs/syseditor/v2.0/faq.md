# 常见问题

## 系统图搭建最佳实践

**Q: 如何搭建高效流畅的系统图？**
A: 为确保系统图的流畅运行，建议遵循以下最佳实践：

1. **控制节点数量**：总节点数建议不超过 10,000 个
2. **优化 SVG 使用**：SVG 节点数量建议不超过 5,000 个
3. **合理使用组合**：优先使用单个图形元素，系统会对单个图形进行缓存优化
4. **限制动画节点**：动画节点（如流动条、旋转元素）数量建议不超过 100 个

## 性能优化

**Q: 系统图编辑时出现卡顿怎么办？**
A: 编辑卡顿通常与节点数量和类型有关，可通过以下方式优化：

1. **清理冗余元素**：删除无用的节点和连接关系
2. **减少 SVG 节点**：SVG 为矢量图，重绘成本较高，建议减少使用
3. **谨慎使用 SVG 图库**：[系统组件-二次接线图]和[系统组件-电力]分类下的图库均为 SVG 节点，建议减少使用频率

**Q: 系统图预览时出现卡顿怎么办？**
A: 预览卡顿主要与渲染性能相关，可通过以下方式优化：

1. **控制动画数量**：将动画节点数量控制在 100 个以内
2. **优化图形结构**：优先使用单个图形元素，避免不必要的组合
3. **控制节点规模**：总节点数不超过 10,000 个，SVG 节点不超过 5,000 个
